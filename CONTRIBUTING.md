# Contributing to PRISM

Good first areas:

- documentation fixes
- README wording
- example extensions
- file type category suggestions
- small bug reports
- testing on Windows/Linux/macOS

Harder areas:

- extension hook design
- config/runtime behavior
- organize/undo safety
- packaging changes

Please ask before changing:

- file movement logic
- undo/log behavior
- extension API contracts
- config format
